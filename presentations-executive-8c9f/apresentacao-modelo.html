Evolução de Vazão & Métricas

Como a equipe do Locavia Automação organizou o funil de entrega e planejou a transição de planilhas para a API do Jira.

Apresentação para Diretoria Executiva • Maio 2026
Cenário Atual

O Desafio dos Dados Manuais

Para visualizar o real gargalo das Squads da Locavia, dependíamos de exportações em massa de planilhas Excel (Base Cone), gerando um fluxo suscetível a erros.

  • Descentralização: Dificuldade de auditar releases.
  • Métricas Mortas: Falta de atualização em tempo real.
  • Ruído de Governança: Cards "Done" sem data de resolução.
03 dias
Tempo Médio de Atraso

Tempo médio gasto em reuniões e alinhamentos para consolidar o status das entregas nas squads devido à falta de automação.

Solução

Novo Dashboard de Vazão (Cone)

Implementamos uma arquitetura robusta de front-end preparada para carregar dados de qualquer origem (Excel ou Jira).

01
Normalização de Dados

Função adaptativa que unifica chaves de data e times, mantendo a estabilidade gráfica mesmo se o formato do JSON mudar.

02
Filtros Avançados

Visualização imediata por Squad, com geração automática de dados históricos e cálculo de vazão acumulada.

03
Auditoria Integrada

Card de alerta que aponta a qualidade dos dados inseridos no Jira, garantindo governança rígida.

Evolução

Plano de Integração Direta (Jira API)

  • Fase 1 (Atual): Sincronização via arquivos estruturados na pasta `/sync`.
  • Fase 2 (Em testes): Geração de tokens de API do Jira.
  • Fase 3 (Q3): Conexão em tempo real do frontend no endpoint do Jira.
Objetivo Final

Reduzir a zero o trabalho operacional de atualização de métricas, permitindo que a diretoria visualize a produtividade da Squad em tempo real, direto no link do dashboard.

Sucesso
